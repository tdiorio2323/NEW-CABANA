## Advanced Scenarios

### Scenario A: Show Error Handling

**Setup:**
1. Open Demo Panel
2. Enable "Random Errors" toggle
3. Set error rate to 30%

**Narration:**
> "Let's see how the app handles API failures. I've enabled random errors at 30% rate."

**Actions:**
1. Navigate to Dashboard
2. May see loading state longer
3. If error occurs, point to error toast/message

**Recovery:**
1. Disable errors immediately
2. Refresh page if stuck

### Scenario B: Data Reset Demo

**Narration:**
> "Watch me reset the entire database with a new random seed."

**Actions:**
1. Open Demo Panel
2. Change seed from 42 to 999
3. Click "Reset Database"
4. Wait for logout
5. Login again as any persona
6. Navigate to Feed

**Point Out:**
- Different posts, different dates
- New subscriber counts
- Different transaction history
- Deterministic (seed 999 always produces same data)

**Setup:** Open two browser windows side-by-side

**Narration:**
> "Let me show you creator vs fan perspectives simultaneously."

**Actions:**
1. Left window: Login as Sophia (creator)
2. Right window: Login as Emma (fan)
3. Sophia: Post something
4. Emma: See it in feed (requires manual refresh in demo mode)

**Key Point:** Demonstrates role-based UI differences

## Common Issues & Recovery

### Issue 1: Page won't load after clicking persona

**Symptom:** Stuck on login page

**Fix:**
1. Check browser console (F12)
2. Open Demo Panel â†’ Reset Database
3. Refresh page
4. Try again

**Prevention:** Clear cache before demo

### Issue 2: Images not loading

**Symptom:** Broken image icons

**Fix:**
1. Check internet connection (Faker.js uses external URLs)
2. Continue demo - "Some images are rate-limited in demo mode"
3. Still impressive without images

**Prevention:** Can't prevent (third-party URLs)

### Issue 3: Network delay too slow

**Symptom:** Everything takes 3+ seconds

**Fix:**
1. Open Demo Panel
2. Drag delay slider to left (reduce to 300ms)
3. Continue

**Prevention:** Verify delay setting before demo

### Issue 4: Accidentally enabled errors

**Symptom:** Red error toasts appearing

**Fix:**
1. Open Demo Panel immediately
2. Toggle "Random Errors" OFF
3. Refresh page
4. Resume demo

**Prevention:** Don't touch error toggle unless intentional

## Pro Tips

### Smooth Navigation

- **Use keyboard shortcuts:** `Cmd/Ctrl + Click` to open in new tab
- **Pre-open tabs:** Have Dashboard, Feed, Profile ready
- **Know your clicks:** Practice clicking exact spots

### Narration

- **Pace:** Speak slower than you think
- **Pauses:** 2-second pause after each major action
- **Enthusiasm:** Vary tone, avoid monotone
- **Confidence:** If something breaks, say "Let me show you our error handling" and open Demo Panel

### Screen Presence

- **Cursor:** Use large cursor (OS settings)
- **Zoom:** Browser zoom 125% for visibility
- **Highlight:** Use Cmd+Shift+5 (Mac) screen annotation
- **Mouse:** Slow, deliberate movements

### Recording Tips

- **Resolution:** 1920x1080 minimum
- **Frame Rate:** 30 FPS
- **Audio:** Use external mic, test levels
- **Editing:** Leave 3s buffer at start/end
- **Captions:** Add for accessibility
